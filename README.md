# Insurance Claims Management Portal

## Project Overview

The Insurance Claims Management Portal is a full-stack portfolio project that models a simplified enterprise claims workflow. The goal is to show how business users, customers, APIs, databases, and administrative processes connect inside an insurance technology environment.

This project is designed to demonstrate business systems thinking, REST API design, database-backed workflows, and user-facing delivery in a portfolio-sized application.

## Project Goals

- Allow users to submit and track insurance claims
- Allow administrators to review, search, and update claims
- Capture claim history for operational transparency
- Practice enterprise application patterns using Spring Boot, PostgreSQL, and React
- Explain technical decisions in business and enterprise terms

## Planned Features

- Submit a new claim
- View claim status
- Upload supporting documentation
- Track claim history
- Admin claim review
- Admin status updates
- Search and filter claims
- Dashboard summary view
- Basic authentication and role-based access

## Tech Stack

- Backend: Java, Spring Boot
- Database: PostgreSQL
- API: REST APIs
- Frontend: React
- UI System: Material Design / Material UI
- Version Control: Git and GitHub

## Portfolio Positioning

This project shows how an insurance workflow can be translated into a digital business system. It is intended for roles such as Business Systems Analyst, Product Analyst, Solutions Analyst, Technology Analyst, and Digital Transformation Analyst.
